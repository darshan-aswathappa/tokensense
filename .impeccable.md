## Design Context

### Project

**TokenSense** — macOS menu bar app (Tauri v2 + React/TypeScript + Rust) for monitoring Claude and Codex token usage in real-time. A compact popover that opens on menu bar click.

### Users

Developers who use Claude AI and Codex heavily. They open the popover for a **2-second glance** to check token budget, then close it. They are in flow state — coding, not browsing. They are comfortable with dense, technical interfaces and expect precision. Information density is respectful; friction is not.

### Brand Personality

**Precise. Disciplined. Transparent.**

Emotional goal: _Control & awareness_ — "I know exactly where I stand." No surprises, no anxiety. The interface delivers the fact and steps aside.

### Aesthetic Direction

- Background `#0a0a0a`, foreground `#f5f5f7`, brand signal `--amber: #ff8c00`
- JetBrains Mono throughout — terminal authenticity, not affectation
- Status color ramp: green (≤70%) → amber (70–90%) → red (≥90%)
- Sharp 2px border-radius everywhere — no rounded cards, no frosted glass
- Dark mode only
- No gradients, no soft shadows, no colored icon decorations

**Anti-references:**

- SaaS dashboards (Notion, Linear, Vercel) — no rounded cards with colored icons, no friendly onboarding
- Activity Monitor — no dense gray utility aesthetic; this should feel _intentionally_ designed

**Reference aesthetic:** A well-designed terminal utility. Data journalism density with developer-tool honesty.

### Design Principles

1. **Glanceability first** — Every piece of information must be readable in under 2 seconds. The user is mid-task; this is a momentary interrupt. Never make them hunt.

2. **Density is respectful** — This audience wants more data, not whitespace. Breathe _between_ sections (sections separated by rules/margins), never _within_ them (no padding inflation inside stat rows or bars).

3. **One signal color, used with restraint** — Amber (`#ff8c00`) marks active state, focus rings, and the 70–90% warning zone. Green/red for status only. Introducing new colors dilutes the signal system.

4. **Terminal authenticity over polish** — Loading states use blinking cursors, not spinners. Empty states read like terminal output. Animations are brief (200–550ms), functional, and respect `prefers-reduced-motion`.

5. **Expandable without sprawl** — Design for 2–4 provider tabs without breaking rhythm. Use hierarchy (size, weight, uppercase labels) to communicate structure, not width or color variety.
